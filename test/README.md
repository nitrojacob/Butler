
# Unit Testing for Butler Project

## Overview
This directory contains unit tests for the Butler project using the Unity testing framework and CMock mocking library. All tests are designed to run on a host machine (Linux/macOS) and do not modify any production firmware code.

## Directory Structure
```
test/
├── README.md
├── unit/              # Unity test files
├── mocks/             # Mock implementations for hardware dependencies
├── unity/             # Unity framework source (downloaded)
├── cmock/             # CMock framework source (downloaded)
└── CMakeLists.txt     # Build configuration for host-based tests
```

## Prerequisites
- GCC compiler (gcc)
- CMake (version 3.16+)
- Git (to clone Unity and CMock)

## Setup Instructions

### 1. Download Unity and CMock
```bash
cd test
git clone https://github.com/ThrowTheSwitch/Unity unity
git clone https://github.com/ThrowTheSwitch/CMock cmock
```

### 2. Build and Run Tests
```bash
cd test
cmake .
make
./test_runner
```

## Test Coverage
- `nvCron.c`: All public functions (`nvCron_traverseList`, `nvCron_writeMultipleEntry`, `nvCron_readMultipleEntry`)
- `tmu.c`: All public functions (`tmu_set`, `tmu_updateRTC`)

## Mocking Strategy
- **NVS**: Simulated using an in-memory hash table (`nvs_mock.c`)
- **System Time**: Mocked using `time_mock.c` to allow deterministic test conditions
- **DS1307**: Mocked using `ds1307_mock.c` to simulate presence/absence of RTC
- **Actuator**: Mocked using `actuator_mock.c` to verify GPIO control calls
- **Task Creation**: Mocked using `freertos_mock.c` to verify `xTaskCreate` calls
